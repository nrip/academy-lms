Mobile Adaptation Notes

Cross-cutting responsive patterns — REQ-ACC-2 (fully responsive, no native app at v1)
1. Try the toggle

The 🖥 Desktop / 📱 Mobile toggle at the top of this sidebar re-flows every screen in this file live — multi-column grids stack to one column, wizard/module step-lists become horizontally-scrollable strips instead of a fixed side rail, and dense tables scroll horizontally rather than clipping. It's a simulation for review purposes, not production breakpoint code, but the patterns it demonstrates are the ones to build against.

2. Pattern by screen type
Screen typeDesktop patternMobile pattern
Multi-step wizard (CA-02)Fixed left step-rail + contentHorizontal step strip on top, full-width single-column content, sticky Back/Next footer
Module/step navigator (L-02, CA-04)Left sidebar listHorizontally scrollable chip row above the content
Dense operational table (R-01, FA-01, F-02)Full multi-column tableHorizontal scroll with a frozen first column (applicant/learner name), or card-per-row for the highest-priority queues
Two/three-column form (CA-02, CA-05, A-06)Side-by-side columnsSingle column, stacked in reading order (primary action fields before secondary summary panels)
Dashboards (L-01, FA-01, F-01)Grid of summary cardsCards stack vertically; "Continue/Resume" card stays first, not buried
Stepper (A-04)Four equal-width segmentsHorizontal scroll, or collapse to "Step 2 of 4: Documents" text on the narrowest widths
3. Deliberately not simulated here

Video player controls, live-session join flows, and file-upload camera capture all have mobile-OS-specific behaviour (native camera roll, picture-in-picture, etc.) that a browser wireframe can't meaningfully approximate — flag these for a real device test pass during build rather than trusting this file for them.
