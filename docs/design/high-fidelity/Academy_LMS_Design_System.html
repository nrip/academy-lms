The Vital Line

A clinical-chart aesthetic for a continuing-education platform: calm, precise, and legible under pressure — because a reviewer verifying a medical registration and a doctor mid-assessment both need the interface to get out of the way. The signature device is a tick-marked measurement line, borrowed from a vitals chart, used everywhere the product shows progress or sequence.

used as: progress bars, wizard steppers, batch-capacity meters

Component Architecture

How this is meant to be built, given the team's Bootstrap + jQuery + HTML5 stack.

  1. Token layer — Bootstrap 5.3's own CSS variables (--bs-primary, --bs-border-radius, --bs-font-monospace, etc.) are retokened in :root, plus a small --acad-* set for the spacing ruler and signature component. No SCSS build step required.
  2. Component layer — default to native Bootstrap classes (btn, form-control, table, card, modal, toast, nav-tabs, badge, alert) styled by the token layer. Minimizes custom CSS surface area and keeps the Bootstrap upgrade path open.
  3. Custom layer — a small acad- prefixed set for the handful of patterns Bootstrap has no equivalent for: acad-pill (status taxonomy), acad-vitalline (signature progress/stepper), acad-card-state (course lifecycle), acad-bubble (internal vs. learner-visible).
  4. Interaction layer — Bootstrap's bundled JS (Popper-based) drives standard components: modal, toast, tabs, dropdowns. jQuery sits alongside it for page-specific behaviour — the section nav on this very page, the Vital Line demo slider, and future flow-specific interactions (stepper logic, live filtering) — rather than fighting Bootstrap's native JS for what it already does well.
  5. Naming convention — acad-{component}, with BEM-style modifiers: acad-pill--success, acad-card-state--scheduled.
